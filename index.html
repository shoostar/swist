OVERVIEW

Mathematically Simple

Everything in SWIST+ is calculated on the fly, from colors, width, and font sizes, to gutters, columns, margins and padding. Now that CSS3 allows for the use of variables without a preprocessor, we can explicitly set variables, and then use them wherever we please using the calc(); function so that we can simply perform the math at run-time.

The codebase is rather fat right now, but I'm working on chunking it down as much as possible, while still retaining some useful features (like the touch-enabled slider). The size of the index.html file will more than likely grow as I add new examples, but I'm going to do my best to trim back the rest. I am also going to include some other shit, like form inputs and generic article/media layouts.

GRID

Headache-free grids

SWIST+ is setup in such a way that the end-user (you) shouldn't have to solve any complicated math equations for sorting out gridlet widths, gutters, padding, or other sizing adjustments. SWIST+ is based on the premise that device width and pixel densities are infinite, but one thing that remains consistent across an individual website is how the grid columns are setup.

Rather trying to figure out the widths and gutters for (admittedly an oddball) 11-column layout, why not express this as columns: 11;, and then for each column, simply assign a width that defines how many columns it spans? Well, now you can. All you need to do is pass the --width: variable along to that gridlet, and the rest is done by your browser.
